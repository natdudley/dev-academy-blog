The difference between absolute and relative positioning

19 November 2010

Position is part of a suite of tools we use to determine how a page is laid out. Absolute and relative are two types of positioning in CSS that determine whether an element appears in normal document flow and what element it is positioned in relation to.

In order to understand how position is used, you need to understand the basics on how a web page behaves by default. This is called ‘normal document flow’. If you already know how normal document flow works, you can skip straight to the Layout section.

Normal document flow

When we’re writing a web page, we add a series of elements to a page, like div, p, img. Every element we add to the page is a rectangular block of pixels.

These elements are added to the page using ‘normal document flow’. There are two ways these blocks can be added:

  • On the same line as the previous element. This is called inline. In-line elements typically add some form of non-text content, like img, svg, and embed, or add semantic meaning to text, like a, time, code, and sup.
  • On a new line. These is called block. Block elements typically add text, like p, h1, li, ul, or document structure, like div, article, section, or footer.

See the Pen Block vs inline by nat (@natdudley) on CodePen.

The direction these are added is determined by the writing mode of the previous element. This defaults to writing horizontally, top to bottom, so unless we change it, inline elements will be added horizontally, and block elements will be added from top to bottom. This is how English and other similar languages work.

It can also be changed so the line direction is vertical, and the blocks added either right to left, or left to right for languages like Japanese.

For the purposes of this post, I’ll describe layout using the default English mode.

Layout

Normal document flow gives us a default layout of our page, but that’s almost always not how we want our web page to appear.

CSS has a range of tools to help us manage the layout and flow of the page. Tools like Flexbox, Grid, and Floats are most commonly used for the heavy lifting of laying out page content. We’re not going to dig into those here, because they’re huge topics on their own! Instead, we’re going to look at another tool, position.

Position

Position determines two things:

  • Does your element live inside or outside of the page flow?
  • Which element is your element positioned in relation to?

Every element on your page will either be positioned or not positioned. By default, every element has the position static. Somewhat confusingly, a position of static means that the element is not positioned. It will appear on the page wherever it is computed to appear in normal flow.

Any element you apply a specific position value to other than static is a positioned element. For any positioned element, you can also specify a top, right, bottom, and left property which will determine its final computed position.

There are a few different types of positioned elements.

Absolutely positioned elements

Absolutely positioned elements are elements which are entirely removed from the normal flow of the page. You can think of them as sitting on another layer, hovering above the page. When you set an absolutely position an element, the elements that come after it in the normal document flow cannot see it any more, and will move their position to fill the gap.

There are two different position values which create an absolute positioned element.

Absolute

Absolute: An element with an absolute value will determine its position based on its closest positioned containing ancestor block and the offsets you specify. That can be hard to wrap your head around, but it basically means that it figures out where it needs to be by looking at all the previous blocks it is inside, and finding the closest one with a position value that isn’t static. If none of its containing blocks has a position, it will base its position on your tag.

See the Pen Absolute position by nat (@natdudley) on CodePen.

Absolute positioning is used for things like setting your navigation bar at the top of the page.

Fixed

Fixed: An element with a fixed position will base its final position on your viewport (which is a fancy name for your browser window), and the offsets you specify. Elements with a fixed position are always visible on the screen.

See the Pen Fixed Position by nat (@natdudley) on CodePen.

Fixed positioning is commonly used if you want your navigation bar to be visible at all times, even when you scroll, or for things like a hovering button to contact customer support teams.

Relatively positioned elements

Relatively positioned elements are elements which still exist in the normal flow of the page. Unlike absolutely positioned elements, relatively positioned elements still impact the position of elements that come after them in the normal document flow. Basically, they still take up space in their original position in the page. A relatively positioned element will compute its position based on its original place in the document flow, and the offsets you specify.

The relative value is used to relatively position an element.

See the Pen Relative positioning by nat (@natdudley) on CodePen.

You might use relative position to alter how two child elements are lined up in a container.

Sticky positioned elements

Sticky positioned elements are a combination of fixed and relative positioning. Sticky elements exist in their relative position until it meets a certain threshold, at which point it becomes fixed. This is used for things like making your navigation bar stick to the top when you scroll.

Summary

By default, elements have a static position in the normal document flow.

You can use the position property to change this. There are 5 possible values for the position property:

  • Static: The default value which is not positioned and leaves element in normal place in normal flow.
  • Relative: A relatively positioned value which leaves the element in normal flow and positions it relative to it’s normal spot.
  • Absolute: An absolutely positioned value which removes the element from normal flow and positions it relative to it’s closest positioned ancestor container.
  • Fixed: An absolutely positioned value which removes the element from normal flow and positions it relative to the viewport.
  • Sticky: A positioned value which acts as a hybrid of relative and sticky.
